Docker Architecture & Workflow:

Docker client connects to Docker daemon
Daemon checks for image locally; if not found, pulls from Docker Hub

Essential Commands Covered:

docker pull - downloads image only
docker run - downloads image and creates container
docker ps / docker ps -a - view running/all containers
docker images - list available images
docker exec -it [container_id] /bin/bash - enter running container
docker start [container_id] - restart stopped container
docker commit [container_id] - convert container to image
docker run --name [name] -d -p [host_port]:[container_port] - run with custom name, detached mode, and port mapping

Docker Scenario-Based Q&A
1. Scenario: Container exits immediately after running

Problem: You run docker run ubuntu but container stops right away
Solution: Ubuntu container has no foreground process. Use docker run -it ubuntu /bin/bash to keep it interactive, or run with a service like docker run -d nginx
2. Scenario: Cannot access application running in container

Problem: Nginx is running in container but can't access via browser
Solution: Use port mapping: docker run -d -p 8080:80 nginx, then access via localhost:8080
3. Scenario: Need to install packages in existing container

Problem: Container is running but missing required tools
Solution:
Enter container: docker exec -it [container_id] /bin/bash
Install packages: apt update && apt install [package]
Save changes: docker commit [container_id] new_image_name
4. Scenario: Too many stopped containers cluttering system

Problem: docker ps -a shows many exited containers
Solution: Remove stopped containers with docker rm [container_id] or use docker container prune
5. Scenario: Want to reuse container with custom name

Problem: Need to identify containers easily
Solution: Use docker run --name myapp -d nginx instead of random names.

Based on the meeting transcript, here are the interview questions and answers discussed:

1. What is Docker?
Answer: Docker is an open-source containerization platform tool that converts application code with binaries and its dependencies, making it lightweight, portable, and environment-friendly. With Docker, we can build, refine and ship images from one environment to another in containerized form.

2. What are containers?
Answer: Running image is known as container.

3. Difference between Docker Pull and Docker Run?
Answer:

Docker Pull: Only downloads the image from Docker Hub
Docker Run: Downloads the image (if not available locally) and also creates and runs the container. It performs both pull and execution.
4. What is Docker Hub?
Answer: Docker Hub is a registry where all the images are maintained. It has around 6.3 million public repositories. We can pull and push images from there. It can be used for personal or professional/company purposes.

5. Docker workflow/architecture - Explain what happens when you run a Docker command?
Answer:

Docker client connects to Docker daemon
Docker daemon checks if image is available locally. If not, it pulls the image from Docker Hub
Docker daemon creates a new container from the image
The container runs and output is visible in the terminal
6. What is Docker daemon?
Answer: Docker daemon is a background process that manages Docker containers and images.

7. Difference between docker run -it and docker exec -it?
Answer:

docker run -it: Used for first time when creating and running a new container in interactive mode
docker exec -it: Used to access an already running container in interactive mode
8. How to convert container to image?
Answer: Use docker commit command with container ID to create an image from a container.

9. What is NGINX?
Answer: NGINX is an open-source web server used for reverse proxy, caching, load balancing, and media streaming. It is a very popular web server with around 15 million applications running on it.
